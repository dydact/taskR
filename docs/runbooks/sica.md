# SICA Self-Critique Runbook

TaskR records SICA sessions to track self-improvement cycles for automation flows.

## Sessions
- Create via `POST /sica/sessions` with `subject_type` (e.g., `flow_run`, `autopm_suggestion`) and `subject_id`.
- Update status/critique/resolution with `PATCH /sica/sessions/{id}`.

## Notes
- Use `POST /sica/notes` to append commentary or operator decisions. Notes are tenant-scoped and tied to a session.

## Workflow
1. Automation failure detected (via events or guardrail halt).
2. Create a SICA session referencing the failing subject.
3. Add critique metadata describing root cause; once remediation is deployed, update `resolution_json` and mark status `resolved`.

## Monitoring
- Subscribe to `preference.guardrail.updated` and `autopm.suggestion.created` events to trigger SICA sessions automatically.
- Reports can be generated by querying `tr_sica_session` and `tr_sica_note` (e.g., open sessions per automation).
